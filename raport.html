Introducere

penGUIniX isi propune sa fie o platforma online de gestiune a unor masini virtuale, oferindui clientului o interfata prietenoasa si acces rapid catre masinile sale.

Comportament

Aplicatia are la baza comunicarea dintre webserver si masinile virtuale/serverele create pentru clienti, prin intermediul unui server proxy. Clientul isi va crea un cont si la cererea acestuia i se vor crea masinile dorite si va primi permisiuni pentru acestea. Altfel, se logheaza in cont, din care poate accesa una din masinile virtuale pentru care are permisiuni si poate rula comenzi pe masina dorita. Comanda dorita este trimisa la masina virtuala, este executata pe aceasta, iar output-ul este trimis inapoi si este afisat pe pagina "terminal".

Gestiune informatii

Informatiile despre useri, masinile virtuale si permisiuni sunt stocate intr-o baza de date. Masinile virtuale/serverele sunt create de catre adminul webserverului, iar logandu-ne intr-un cont "admin" ii vom putea oferi clientului permisiuni pentru masina virtuala creata. Permisiunile clientilor pentru masinile virtuale sunt stocate intr-o tabela cu 2 coloane(id_user si id_machine).

Comunicarea dintre webserver si masinile virtuale s-a realizat folosind agenti, masinile virtuale initiind un tunel ssh la pornire care aduce portul acesteia pe proxy.

Tehnologii

Pentru serverul web am folosit HTML, CSS si PHP.

Agentii si serverul proxy au fost scrisi in C si s-a folosit ssh.
